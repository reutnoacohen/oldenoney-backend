# עדכון בפרונט (Contact.tsx) – ליד כפול

ה-backend מחזיר **409 (Conflict)** כשנשלח ליד עם **אותו אימייל + אותו טלפון** שכבר קיים.

## מה לשנות ב-`handleSubmit` ב-Contact.tsx

אחרי שקוראים את `data` מהתשובה (מהטקסט / מ־JSON), **לפני** או **בתוך** הבלוק של `else` (כש־`!response.ok`):

1. **להציג את הודעת השגיאה מהשרת** (כבר מומלץ קודם):
   ```ts
   description: data?.error || data?.message || "שגיאה בשליחת ההודעה. נסה שוב מאוחר יותר.",
   ```

2. **לטפל ב-409 (ליד כפול)** – אם את רוצה הודעה ייעודית כשהליד כבר נשלח:
   - אפשר לבדוק `response.status === 409` ולהוציא toast עם הכותרת "ליד כפול" וה־`description` = `data?.error` (השרת שולח: "ליד עם אימייל וטלפון אלה כבר נשלח. לא ניתן לשלוח שוב.").
   - או פשוט להשאיר את אותה לוגיקה של `!response.ok` עם `data?.error || data?.message` – אז המשתמש יראה אוטומטית את ההודעה מהשרת גם ב-409.

**דוגמה (אם את רוצה טיפול מפורש ב-409):**
```ts
} else {
  const isDuplicate = res.status === 409;
  toast({
    title: isDuplicate ? "ליד כפול" : "שגיאה",
    description: data?.error || data?.message || "שגיאה בשליחת ההודעה. נסה שוב מאוחר יותר.",
    variant: "destructive",
  });
}
```

או בלי להבדיל – רק וידוא ש־`description` מגיע מ־`data?.error || data?.message` ואז גם ב-409 תופיע ההודעה "ליד עם אימייל וטלפון אלה כבר נשלח. לא ניתן לשלוח שוב."
